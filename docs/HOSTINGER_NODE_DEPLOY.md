# How to Deploy the Node.js App to Hostinger

This app must be deployed as a **Node.js Web App** (not a static site). Follow these settings so the site routes correctly to the Node app.

## Hostinger panel checklist

1. **Application type**
   - Select **Node.js Web App** (not "Static site" or "PHP").

2. **Node.js version**
   - Set to **18.x** (e.g. 18.20 or latest 18). The project `package.json` specifies `"engines": { "node": ">=18.0.0" }`.

3. **Application root**
   - Point to the folder that contains `package.json` (repository root). Do **not** point to `public` or `public_html` as the app root.

4. **Build & start commands**
   - **Install:** `npm ci` (or `npm install`)
   - **Build:** `npm run build`
   - **Start:** `npm run start` (runs `next start`; Hostinger sets `PORT` automatically)

5. **public_html**
   - After redeploying, `.htaccess` in `public_html` should be **auto-generated by Hostinger** (used to proxy requests to the Node app).
   - Ensure **no other files** are in `public_html` (no old static export files). The live site is served by the Node process, not from files in that folder.

## package.json requirements (already set)

- **Scripts:** `build` and `start` are required. This repo has:
  - `"build": "next build --webpack"`
  - `"start": "next start"`
- **Dependencies:** `next`, `react`, `react-dom` and other runtime deps are in `dependencies` (not only in devDependencies).

## Environment variables (optional)

Set in Hostinger’s Node.js app environment if needed:

- `NODE_ENV=production`
- `NEXT_PUBLIC_WORDPRESS_API_URL=https://m.gtmstack.pro/wp-json/wp/v2` (for blog)

## After deployment

- The app listens on the port Hostinger provides via `PORT`.
- Do **not** use `npm run build:static` or `STATIC_EXPORT=1` for this setup; that builds a static export and is for a different hosting mode.
